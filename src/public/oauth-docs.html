Autorizar

Página de login del SSO. El cliente redirige al navegador del usuario a esta URL con los parámetros como query string. El usuario ingresa sus credenciales y, tras autenticarse, la UI hace callback al redirect_uri registrado con el code.

🔒 PKCE — Cómo generar el code_challenge

Antes de redirigir, genera un code_verifier aleatorio y deriva el code_challenge con SHA-256 + Base64url:
JavaScript — Generar PKCE
// 1. Generar code_verifier aleatorio
const array = crypto.getRandomValues(new Uint8Array(32));
const codeVerifier = btoa(String.fromCharCode(...array))
  .replace(/\+/g, '-').replace(/\//g, '_').replace(/=/g, '');

// 2. Derivar code_challenge = BASE64URL(SHA256(code_verifier))
const encoded = new TextEncoder().encode(codeVerifier);
const digest  = await crypto.subtle.digest('SHA-256', encoded);
const codeChallenge = btoa(String.fromCharCode(...new Uint8Array(digest)))
  .replace(/\+/g, '-').replace(/\//g, '_').replace(/=/g, '');

// Guarda codeVerifier — lo necesitas después en POST /token
sessionStorage.setItem('pkce_verifier', codeVerifier);
Query ParamTipoDescripción
client_idrequired string Identificador del cliente OAuth registrado en el servidor.
staterequired string Valor aleatorio opaco para prevenir CSRF. Se devuelve intacto en el callback. Generar con crypto.randomUUID().
code_challengerequired string Hash BASE64URL(SHA-256(code_verifier)). Ver snippet de generación arriba.
URL completa
GET https://sso-ui-delta.vercel.app/authorize?client_id=mi-app-client-id&state=abc123xyz&code_challenge=E9Melhoa2OwvFrEMTJguCHaoeK1t8URWbuGJSstw-cM
JavaScript — Redirigir al navegador
const params = new URLSearchParams({
  client_id:      'mi-app-client-id',
  state:          state,           // guárdalo para validar en el callback
  code_challenge: codeChallenge,
});

window.location.href =
  `https://sso-ui-delta.vercel.app/authorize?${params}`;

Tras el login exitoso, la UI redirige al redirect_uri registrado del cliente:

Callback recibido en redirect_uri
https://mi-app.com/callback?code=SplxlOBeZQQYbYS6WxSbIA&state=abc123xyz
⚠ Valida el state — Verifica que el state del callback coincide con el que guardaste antes de redirigir. Si no coincide, descarta la respuesta (posible ataque CSRF).

https://sso-itg.fly.dev/oauth/token — Authorization Code

Code

Intercambia un authorization code (obtenido del callback de GET /authorize) por un access_token y refresh_token. Se debe enviar el code_verifier original para que el servidor verifique el PKCE.

📋 Content-Type: Los parámetros se envían como application/x-www-form-urlencoded, no como JSON.
CampoTipoDescripción
grant_typerequired string Debe ser authorization_code.
coderequired string El authorization code recibido en el callback de la UI.
code_verifierrequired string El code_verifier original generado antes de iniciar el flujo PKCE.
redirect_urirequired string (URL) Debe ser idéntica al redirect_uri registrado del cliente.
client_idrequired string Identificador del cliente OAuth.
client_secretrequired string Secreto del cliente.
HTTP · application/x-www-form-urlencoded
POST https://sso-itg.fly.dev/oauth/token
Content-Type: application/x-www-form-urlencoded

grant_type=authorization_code&code=SplxlOBeZQQ...&code_verifier=dBjftJeZ4CVP...&redirect_uri=https%3A%2F%2Fmi-app.com%2Fcallback&client_id=sso:xxx:web:xxx&client_secret=73c31507...
cURL

client_idrequired string Identificador del cliente OAuth.
client_secretrequired string Secreto del cliente.

Revok
